[Отчёт](./report.docx) находится в хранилище Git LFS, можно загрузить когда угодно.

## Об игре
1. Есть `n`-нное количество игроков, ходят они по очереди в рамках одного хода.
2. После броска кубика выпадает `m`, это есть число клеток на сколько стоит переместиться игроку.

Игрок может попасть в данные клетки:
|Название|Функция|
|:--|:--|
|Обычная|Является обычной клеткой, ничего не происходит при попадании на неё|
|Ловушка|При попадании на неё игрок обязан пропустить этот ход, то есть пропустить игрока вперёд|
|Бонус|При попадании на неё игроку даётся дополнительный ход|
|Телепорт|При попадании на неё игрок телепортируется на клетку, которая в нём указана|
|Финиш|Игрок, попадая на неё первым выигрывает|

Все клетки описаны в подмодуле `cell` модуля `models` крейта `core`.

## Архитектура

Основная библиотека и логика располагается в крейте `core`. Есть три структуры: `Cell`, `Game` и `Player`.

- `Player` имеет функцию для того, чтобы сходить на рандомное количество шагов (от 1 до 6).
- `Cell` имеет функцию для того, чтобы при попадении на определённую клетку на игрока накладывался опред. эффект. Также здесь решается менять игрока или нет и перемещение самого игрока при попадении на клетку телепорта.
- `Game` - сам объект игры, имеет игроков и клетки, также хранит состояние о том, кто ходит.

## Платформы

Тестировалось на Windows 11 26H1 и NixOS 26.05 (unstable ветка).

Пойти может на любой современной операционной системе, для Windows это Windows 10 и новее, для Linux это glibc 2.30 и новее, версия ядра Linux не особо важна.

Для musl и иных платформ (*BSD, MacOS) говорить ничего не могу, проверяйте сами

## Развёртка

### Компиляция из исходников

1. Для начала требуется склонировать этот репозиторий с помощью `git clone git@github.com:Urtyom-Alyanov/Walkers`
2. После клонирования, убедитесь что у вас установлена версия (или выше) компилятора, указанная в [инструментах разработки](#Инструменты-разработки). После того как вы убедились в том, что у вас нужная версия, можно скомпилировать и запустить этот проект:
```shell
cargo run
```
